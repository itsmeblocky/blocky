quickstart.lua 
local blocky = loadstring(game:HttpGet("URL"))()

local window = blocky.new("My Script", {
    toggleKey = Enum.KeyCode.RightShift,
    accentColor = Color3.fromRGB(0, 120, 215)
})

local tab = window:addTab("Main")
window:addSection(tab, "Controls")

window:addButton(tab, "Click Me", "Description", function()
    print("Hello from Blocky!")
end)

window:addToggle(tab, "Auto Farm", false, function(state)
    print("Toggle:", state)
end)

window:addSlider(tab, "Speed", {
    min = 0, max = 100,
    default = 16, suffix = " studs"
}, function(v) print(v) end)
Features

What's Included

Every feature built into Blocky, extracted directly from source.

WIN

Draggable & Resizable Windows

Full window management with title bar dragging and 8-directional edge resizing. Minimum size enforcement (380×280) with smooth repositioning.

Core
TAB

Tabbed Navigation

Horizontal tab bar with animated underline indicators. Auto-selects first tab. Each tab has its own scrollable content area with auto-sizing canvas.

Core
TWN

Tween Animations

Three tween speeds — fast (0.15s), medium (0.25s), slow (0.35s) — all using Quad easing. Every hover, click, and state change is smoothly animated.

Core
ACC

Runtime Accent Theming

Call window:setAccent(color) to change accent color across all elements at runtime. Internal listener system propagates changes automatically.

Event
EVT

Global Event System

13 named events via blocky.on(name, callback) — from button clicks and toggle changes to keybind fires and window state changes.

Event
NTF

Notification System

Toast-style popups with 4 types — info, success, error, warning — each with its own accent color, icon, progress bar, and configurable duration.

Core
MIN

Minimize & Close

Title bar buttons for minimizing (collapses to title bar height with animation) and closing (shrinks to zero then destroys). Both fire events.

Core
TGL

Toggle Key Visibility

Set an optional toggleKey to show/hide the entire window with a single keypress. Respects gameProcessedEvent to avoid conflicts.

Core
THM

13-Color Theme System

BASE_THEME defines Background, Secondary, Tertiary, Active, Accent, Text, SubText, Error, Success, Warning, Border, TitleBar, and Console colors.

Core
CLR

HSV Color Picker

Full color picker with hue wheel (ring), saturation/value square, draggable knobs, and hex input. Opens as a floating panel anchored to the element.

Core
KEY

Rebindable Keybinds

Click to enter listening mode, press any key to bind. Fires callback when the bound key is pressed during gameplay. Displays current key in bracket notation.

Core
RET

Return Objects with :set()

Toggle, Slider, Dropdown, and ColorPicker all return objects with a .value property and a :set() method for programmatic control.

Events

13 Global Events

Register listeners with blocky.on(name, callback)

"notifyFired"
(title, message, type, duration)
Fires when a notification is shown
"windowClosed"
(title)
Fires when X button is clicked
"windowMinimized"
(title, minimized)
Fires on minimize/restore toggle
"windowToggled"
(title, visible)
Fires when toggleKey shows/hides window
"tabSwitched"
(name)
Fires when a tab button is clicked
"buttonClicked"
(label)
Fires when any button element is clicked
"toggleChanged"
(label, state)
Fires when a toggle's state changes
"dropdownChanged"
(label, selected)
Fires when a dropdown selection changes
"inputSubmitted"
(label, text)
Fires when Enter is pressed in a text input
"keybindChanged"
(label, key)
Fires when a keybind is rebound to a new key
"keybindFired"
(label, key)
Fires when a bound key is pressed during gameplay
"colorChanged"
(label, color)
Fires when a color picker value changes
"accentChanged"
(color)
Fires when setAccent() changes the accent color

API Reference

Complete Method Reference

Every public method extracted from the source code.

blocky.new(title, options?)
Creates a new window. Returns the window object.
title — string, title bar text
options.toggleKey — Enum.KeyCode?
options.accentColor — Color3?
→ window { :addTab, :setAccent, :onAccentChanged }
blocky.notify(title, msg, type?, dur?)
Shows a toast notification popup.
type — "info" | "success" | "error" | "warning"
duration — number (default: 3)
→ void
blocky.on(eventName, callback)
Registers a global event listener.
eventName — one of 13 event names
callback — function
→ void
window:addTab(name)
Adds a tab to the tab bar with scrollable content.
name — string, tab label
→ tab (pass to add* methods)
window:addButton(tab, label, desc?, cb?)
Adds a clickable button row with flash effect.
desc — optional subtitle string
cb — function, called on click
→ TextButton
window:addToggle(tab, label, desc?, def?, cb?)
Adds a checkbox toggle. Flexible argument order.
default — boolean (default: false)
cb — function(state: boolean)
→ { value: boolean, :set(boolean) }
window:addSlider(tab, label, desc?, config, cb?)
Adds a draggable slider with value display.
config.min / .max / .default / .suffix
cb — function(value: number)
→ { value: number, :set(number) }
window:addDropdown(tab, label, desc?, opts, def?, cb?)
Adds a dropdown selector with animated expand.
opts — {string}, list of options
cb — function(selected: string)
→ { value: string, :set(string) }
window:addInput(tab, label, desc?, placeholder?, cb?)
Adds a text input that fires on Enter.
placeholder — hint text
cb — function(text: string)
→ TextBox
window:addKeybind(tab, label, desc?, def?, cb?)
Adds a rebindable keybind with listening mode.
default — Enum.KeyCode?
cb — function(), called on keypress
→ TextButton
window:addColorPicker(tab, label, desc?, def?, cb?)
Adds a color picker with hue wheel + SV square.
default — Color3?
cb — function(color: Color3)
→ { value: Color3, :set(Color3) }
window:addSection(tab, label)
Adds a section header with accent top-line animation.
label — section heading text
→ Frame
window:addLabel(tab, text)
Adds a read-only monospace label row.
text — display text
→ TextLabel
window:addSeparator(tab)
Adds a 1px horizontal divider line.
No parameters
→ Frame
window:setAccent(color)
Changes accent color at runtime across all elements.
color — Color3
→ void (fires "accentChanged" event)
window:onAccentChanged(callback)
Internal listener for accent color changes.
callback — function(color: Color3)
→ void

FAQ

Common Questions

What elements are included? +
12 elements: addButton, addToggle, addSlider, addDropdown, addInput, addKeybind, addColorPicker, addSection, addLabel, addSeparator, plus the Window and Notification system.
What's the difference between the Devs and Exploiters version? +
The Devs version is designed for Roblox Studio — you can require it as a ModuleScript. The Exploiters version is optimized for executor environments and supports loadstring directly. Both expose the same full API with all 12 elements and 13 events.
Can I change the accent color at runtime? +
Yes. Call window:setAccent(Color3.fromRGB(r, g, b)) and all accent-colored elements — toggle checkboxes, slider fills, tab underlines, section top lines, and slider value labels — update automatically via the internal onAccentChanged listener system. This also fires the global "accentChanged" event.
Can I programmatically control element values? +
Yes. addToggle, addSlider, addDropdown, and addColorPicker all return objects with a .value property and a :set(newValue) method. For example: myToggle:set(true) or mySlider:set(50).
How does the color picker work? +
Clicking the color preview swatch opens a floating picker anchored to the element. It contains a hue ring (circular wheel), a saturation/value square in the center, draggable knobs for both, and a hex code input box at the bottom. Changes fire the callback and the "colorChanged" event.
Are the windows resizable? +
Yes. Windows support 8-directional edge resizing — Top, Bottom, Left, Right, and all four corners. Edge detection uses a 6px threshold. Minimum size is enforced at 380×280 pixels. The title bar is used for dragging.
What notification types are available? +
Four types: "info" (blue, [i]), "success" (green, [+]), "error" (red, [-]), and "warning" (yellow, [!]). Each gets its own accent color bar, icon, and progress bar. Default duration is 3 seconds.
How do keybinds work? +
Click the key display button to enter listening mode (shows "[...]" with accent color). Press any keyboard key to bind it. The display updates to show [KeyName]. When that key is pressed during gameplay (not when typing), the callback fires. The "keybindChanged" event fires on rebind, and "keybindFired" fires on activation.
